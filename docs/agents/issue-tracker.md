# Issue tracker: GitHub

Issues and PRDs for this repo live as GitHub issues. Use the `gh` CLI for all operations.

Repository: `awjreynolds/banes-satn`

## Conventions

- Create an issue with `gh issue create`.
- Read issues and comments with `gh issue view`.
- List issues with `gh issue list`.
- Comment with `gh issue comment`.
- Apply or remove labels with `gh issue edit`.
- Close issues with `gh issue close`.
- Infer the repository from the GitHub remote.
- Pull requests are not a triage request surface.

## Skill conventions

- “Publish to the issue tracker” means create a GitHub issue.
- “Fetch the relevant ticket” means read the complete GitHub issue and comments.
- Wayfinder maps and child tickets use GitHub issues.
- Use native sub-issue and blocking relationships when available.
- Fall back to explicit task lists and `Blocked by:` references when necessary.
